# 날씨 앱 (Weather App)

React와 TypeScript를 활용한 날씨 정보 조회 애플리케이션입니다.

## 프로젝트 실행 방법

### 1. 의존성 설치

```bash
npm install
```

### 2. 환경 변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 OpenWeatherMap API 키를 설정하세요:

```env
VITE_WEATHER_API_KEY=your_openweathermap_api_key_here
```

OpenWeatherMap API 키는 [OpenWeatherMap](https://openweathermap.org/api)에서 무료로 발급받을 수 있습니다.

### 3. 행정구역 데이터 파일 준비

`public` 폴더에 `korea_districts.json` 파일을 추가하세요. 이 파일은 대한민국 행정구역 정보를 포함해야 하며, 다음과 같은 구조를 가져야 합니다:

```json
[
  {
    "시도명": "서울특별시",
    "시군구명": "종로구",
    "읍면동명": "청운동",
    "경도": 126.9667,
    "위도": 37.5888
  }
]
```

### 4. 개발 서버 실행

```bash
npm run dev
```

### 5. 프로덕션 빌드

```bash
npm run build
```

빌드된 파일은 `dist` 폴더에 생성됩니다.

## 구현한 기능

### 1. 날씨 정보 조회

- OpenWeatherMap API를 사용하여 날씨 정보를 가져옵니다
- 현재 기온, 당일 최저/최고 기온, 시간대별 기온을 표시합니다
- 날씨 아이콘과 설명을 함께 표시합니다

### 2. 현재 위치 감지

- 앱 첫 진입 시 사용자의 현재 위치를 자동으로 감지합니다
- Geolocation API를 사용하여 위도/경도를 가져옵니다
- 위치 정보가 없는 경우 검색 기능을 사용할 수 있습니다

### 3. 장소 검색

- 시, 군, 구, 동 모든 단위로 검색 가능합니다
- 예: "서울특별시", "종로구", "청운동" 등
- 검색어 입력 시 매칭되는 장소 리스트를 실시간으로 표시합니다
- 검색 결과에서 장소를 선택하면 해당 위치의 날씨 정보를 조회합니다
- 날씨 정보가 없는 경우 "해당 장소의 정보가 제공되지 않습니다." 메시지를 표시합니다

### 4. 즐겨찾기 기능

- 검색한 장소를 즐겨찾기에 추가/삭제할 수 있습니다
- 최대 6개의 장소를 즐겨찾기에 추가할 수 있습니다
- 즐겨찾기 장소는 카드 UI 형태로 표시됩니다
- 즐겨찾기 장소의 별칭을 수정할 수 있습니다
- 즐겨찾기 카드에는 현재 날씨 정보와 당일 기온(최저, 최고)이 표시됩니다
- 즐겨찾기 카드를 클릭하면 해당 장소의 상세 페이지로 이동합니다
- 상세 페이지에는 모든 날씨 정보(현재 기온, 최저/최고 기온, 시간대별 기온)가 표시됩니다

## 기술적 의사결정 및 이유

### 1. FSD (Feature Sliced Design) 아키텍처

- **이유**: 코드의 가독성과 유지보수성을 높이기 위해 FSD 아키텍처를 채택했습니다
- **구조**:
  - `app/`: 앱 초기화 및 라우팅
  - `pages/`: 페이지 레벨 컴포넌트
  - `widgets/`: 복합 위젯 (날씨 카드)
  - `features/`: 기능 단위 (날씨 표시, 위치 검색, 즐겨찾기)
  - `entities/`: 비즈니스 엔티티 (위치, 날씨)
  - `shared/`: 공유 모듈 (API, 유틸리티, UI 컴포넌트)

### 2. Tanstack Query (React Query)

- **이유**: 서버 상태 관리와 캐싱을 효율적으로 처리하기 위해 사용했습니다
- **장점**:
  - 자동 캐싱 및 리페칭
  - 로딩 및 에러 상태 관리
  - 중복 요청 방지
  - 5분간 데이터를 캐시하여 불필요한 API 호출 감소

### 3. React Router

- **이유**: SPA에서 페이지 간 이동을 구현하기 위해 사용했습니다
- **구현**: 홈, 상세 페이지, 즐겨찾기 페이지 간 라우팅

### 4. Tailwind CSS

- **이유**: 빠른 UI 개발과 일관된 디자인 시스템을 위해 사용했습니다
- **장점**:
  - 유틸리티 기반 CSS로 빠른 스타일링
  - 반응형 디자인을 쉽게 구현
  - 커스텀 컴포넌트와 조합하여 재사용성 향상

### 5. localStorage를 활용한 즐겨찾기 관리

- **이유**: 클라이언트 사이드에서 간단하게 데이터를 저장하기 위해 사용했습니다
- **구현**: 즐겨찾기 추가/삭제/수정 기능을 localStorage에 저장

### 6. TypeScript

- **이유**: 타입 안정성과 개발 생산성 향상을 위해 사용했습니다
- **장점**:
  - 컴파일 타임에 오류 발견
  - 자동완성 및 IntelliSense 지원
  - 코드 가독성 향상

### 7. OpenWeatherMap API

- **이유**: 무료로 제공되며 안정적인 날씨 데이터를 제공하기 때문에 선택했습니다
- **사용 API**: One Call API 3.0 (현재 날씨, 일일 예보, 시간별 예보)

## 사용한 기술 스택

### Core

- **@tanstack/react-query**: 서버 상태 관리 및 캐싱
- **@tanstack/react-query-devtools**: 개발 도구

### 라우팅

- **react-router-dom**: 클라이언트 사이드 라우팅

### 스타일링

- **Tailwind CSS**: 유틸리티 기반 CSS 프레임워크

- **axios**: HTTP 요청 라이브러리

### 아키텍처

- **FSD (Feature Sliced Design)**: 프로젝트 구조 설계 방법론

## 프로젝트 구조

```
src/
├── app/                    # 앱 초기화
│   ├── providers/         # 전역 프로바이더
│   └── router.tsx         # 라우팅 설정
├── pages/                  # 페이지 레벨
│   ├── home/              # 홈 페이지
│   ├── location-detail/   # 위치 상세 페이지
│   └── favorites/         # 즐겨찾기 페이지
├── features/              # 기능 단위
│   ├── weather-display/   # 날씨 표시
│   ├── location-search/   # 위치 검색
│   └── favorites/         # 즐겨찾기
├── entities/              # 비즈니스 엔티티
│   ├── location/          # 위치 엔티티
│   └── weather/           # 날씨 엔티티
├── shared/                # 공유 모듈
│   ├── api/               # API 클라이언트
│   ├── lib/               # 유틸리티
│   ├── ui/                # 공통 UI 컴포넌트
│   └── hooks/             # 공통 훅
└── widgets/               # 복합 위젯
    └── weather-card/      # 날씨 카드 위젯
```

## 반응형 디자인

- **모바일**: 320px 이상
- **태블릿**: 768px 이상 (md)
- **데스크탑**: 1024px 이상 (lg)

Tailwind CSS의 반응형 유틸리티 클래스를 사용하여 모든 화면 크기에서 최적화된 UI를 제공합니다.

## 배포

이 프로젝트는 Vercel, Netlify 등 정적 호스팅 서비스에 배포할 수 있습니다.

```bash
npm run build
```

빌드 후 `dist` 폴더의 내용을 배포 서비스에 업로드하세요.
